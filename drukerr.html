UX Case Study

Duration: 7th May - 10th May

Product Overview

DrukErr is a responsive platform designed for users in Bhutan to post technical or real-life problems and receive community-driven answers, similar to Stack Overflow. The website is designed to work well on laptops and mobile devices. It offers features such as categorized Q&A, search functionality, upvoting, and user discussions—making it a one-stop solution for problem-solving and collaborative knowledge-sharing.


Problem

People often face difficulties when looking for reliable answers to their technical or real-life problems because solutions are scattered across multiple platforms, or lack localized and contextual relevance. This results in wasted time, duplicate queries, and limited community engagement.


Goal

The platform aims to be a centralized, accessible, and responsive space where users can post problems across categories, receive community answers, and upvote the best responses. With powerful search and filter features, users can easily find relevant content, while structured, high-quality contributions foster collaborative learning.

Accessibility

Scalable Fonts for better readability

  • I implemented adjustable text sizes and clear, legible fonts to accommodate users with varying levels of visual acuity.

    • • Keyboard Navigation for users with mobility issues

  • I ensured that all interactive elements—such as buttons, links, and form inputs—are accessible via the keyboard using the Tab, Enter, and Arrow keys.

    • Consistent Structure Across All Pages

  • I implemented a reusable layout with consistent headers, navigation bars, and footers across all pages. This structure helps users—especially those with cognitive impairments—predict where to find information and navigate confidently.
